Dépôt (Repository) : Espace de stockage d’un projet contenant les fichiers et l’historique des modifications.

Commit : Enregistrement d’une modification dans le dépôt, accompagné d’un message décrivant le changement.

Branche (Branch) : Version parallèle du projet permettant de développer une fonctionnalité sans modifier la branche principale.

Merge : Action de fusionner les modifications d’une branche dans une autre.

Pull Request (PR) : Demande de fusion d’une branche vers une autre, permettant la relecture et la discussion avant l’intégration.

README.md : Fichier de documentation principal d’un projet, expliquant son objectif, son installation et son utilisation.

.gitignore : Fichier qui liste les fichiers ou dossiers que Git doit ignorer (ex. : fichiers temporaires).

Repository visibility (public/private) : Paramètre qui définit si le dépôt est accessible à tous (public) ou uniquement aux personnes autorisées (privé).

PAT (Personal Access Token) : Clé d’authentification personnelle utilisée pour accéder à un dépôt ou à une API de manière sécurisée.

Protection de branche : Règles appliquées à une branche pour empêcher des modifications directes ou imposer des validations avant un merge.
